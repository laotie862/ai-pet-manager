## 一、技术可行性

### 1.1 视频解码能力
| 技术点 | 方案 | 成熟度 |
|--------|------|--------|
| 视频解码 | FFmpeg / JavaCV | 业界标准，成熟稳定 |
| 格式支持 | MP4/AVI/MKV/FLV | 全面覆盖主流格式 |
| 网络流拉取 | yt-dlp + FFmpeg | 已验证，可获取YouTube/B站直播流 |
| 循环播放 | FFmpeg stream_loop | 原生支持，-1为无限循环 |
| 倍速播放 | FFmpeg setpts滤镜 | 原生支持，0.5x~16x |
| 逐帧提取 | OpenCV VideoCapture | 每帧30ms，性能足够 |

### 1.2 与现有CV模块的兼容性
```
现有CV模块输入：摄像头帧 (Mat/BufferedImage)
视频模拟输出：  逐帧画面 (Mat/BufferedImage)
结论：接口完全一致，无需改动CV模块代码
```

| 检查项 | 结果 |
|--------|------|
| 输入格式 | 兼容（均为按帧输入的图像数据） |
| 帧率 | 可配置，匹配原25fps |
| 分辨率 | 可统一预处理为640x480 |
| 时间戳 | 可映射虚拟时间 |
| ROI区域 | 完全复用现有逻辑 |

## 二、经济可行性

| 对比项 | 物理摄像头方案 | 视频模拟方案 |
|--------|---------------|-------------|
| 硬件采购 | 3-5台摄像头 ≈ 1500-5000元 | 0元 |
| 环境搭建 | 宠物寄养/志愿者 ≈ 2000元/月 | 0元 |
| 测试周期 | 1:1实时，约2周 | 可加速，约3天 |
| 维护成本 | 设备故障/网络问题 | 纯软件，低维护 |
| **Demo阶段总成本** | **约5000-10000元** | **约0元** |
