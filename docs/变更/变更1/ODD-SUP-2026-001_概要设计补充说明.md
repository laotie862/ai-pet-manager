---

## 一、设计原则
1. 不修改现有 device 模块的公开接口
2. 虚拟视频源实现与物理摄像头相同的帧输出接口
3. 通过配置切换物理/虚拟模式
4. 支持本地文件 + 网络URL两种视频来源

## 二、新增模块结构
```
videosource/
├── VideoSourceType.java        # 枚举：LOCAL_FILE, NETWORK_URL, YOUTUBE_LIVE
├── VideoSourceConfig.java      # 视频源配置
├── VideoSourceRecord.java      # 数据实体
├── VideoSourceRepository.java  # 数据访问层
├── VirtualStreamProvider.java  # 视频→逐帧输出
├── SimulatedClockService.java  # 虚拟时钟映射
├── VideoSourceController.java  # REST接口
└── VideoSourceService.java     # 业务逻辑层
```

## 三、核心接口
VirtualStreamProvider: open/readFrame/seekFrame/setSpeed/isLooping/close
SimulatedClockService: startSimulation/getCurrentVirtualTime/pause/resume/setTimeScale

## 四、设备类型扩展
DeviceRecord.sourceType: RTSP(0), LOCAL_WEBCAM(1), VIDEO_FILE(2), VIDEO_URL(3)

## 五、数据库变更
```sql
CREATE TABLE video_source (
    id UUID PRIMARY KEY,
    device_id UUID REFERENCES device(id),
    source_type VARCHAR(20) NOT NULL,
    source_path TEXT NOT NULL,
    loop_enabled BOOLEAN DEFAULT true,
    speed_factor DOUBLE PRECISION DEFAULT 1.0,
    target_fps INTEGER DEFAULT 25,
    status VARCHAR(20) DEFAULT 'STOPPED'
);
```
