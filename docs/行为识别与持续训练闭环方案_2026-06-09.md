# 行为识别与持续训练闭环方案

日期：2026-06-09

## 1. 结论

项目核心从“先接真实摄像头”调整为“先跑通视频中的行为识别”。短期用本地视频循环模拟摄像头流，对视频抽帧后调用 AI 视觉 API 识别宠物行为，并把识别结果沉淀成训练数据。后期再接真实 RTSP 摄像头，并逐步训练本地模型替代外部 API。

推荐路线：

```text
视频循环/摄像头流
  -> 抽帧
  -> AI API 识别行为
  -> 返回固定标签和置信度
  -> 后端保存行为事件/摘要
  -> 图片与标签沉淀为训练候选集
  -> 人工确认低置信度样本
  -> 定期训练本地模型
  -> 本地模型逐步替代 AI API
```

## 2. 为什么这样做

真实摄像头接入主要解决“视频从哪里来”，但本项目的核心价值是“从视频里判断宠物在做什么”。因此第一阶段用循环视频模拟摄像头流，可以更快验证行为识别、事件落库、摘要统计和前端展示这条主链路。

AI API 适合作为前期“老师模型”：

```text
1. 不需要一开始就准备大量标注数据
2. 可以快速得到 eating/drinking/sleeping 等行为标签
3. 可以把识别过的图片沉淀成本地训练数据
4. 后期本地模型成熟后再降低 API 依赖
```

不建议每识别一帧就立刻训练一次模型，因为错误标签会被模型反复学习，导致越训练越偏。更稳的方式是“识别、采样、确认、批量训练”。

## 3. 行为标签

第一版固定为 6 类：

```text
eating      吃饭
drinking    喝水
exercising  活动/运动
sleeping    睡觉
defecating  排泄
uncertain   不确定
```

所有 provider 都必须只返回这些标签之一，避免后端事件类型发散。

## 4. 模块职责

后端：

```text
1. 管理宠物、设备、视频源
2. 从 DeviceStreamManager 获取最新 JPEG 帧
3. 调用 cv-service /cv/detect
4. 根据稳定帧规则创建/关闭行为事件
5. 保存行为摘要
6. 后续保存抽帧样本和人工确认结果
```

cv-service：

```text
1. 接收 image_base64、device_id、roi_polygon
2. 按 CV_PROVIDER 选择识别方式
3. 返回 behavior、confidence、boxes、model_version
4. 后续负责样本归档和本地模型加载
```

数据集：

```text
dataset/raw-videos/              原始视频
dataset/extracted-frames/        抽帧缓存
dataset/behavior-classifier/     分类训练集
```

## 5. Provider 设计

CV 服务保持同一个接口，不同识别方式通过配置切换：

```text
CV_PROVIDER=placeholder  使用本地占位分类器，方便联调
CV_PROVIDER=bailian      调用阿里云百炼视觉 API 识别抽帧图片
CV_PROVIDER=local        使用本地训练好的模型推理
```

响应格式保持不变：

```json
{
  "found": true,
  "behavior": "eating",
  "confidence": 0.82,
  "boxes": [],
  "model_version": "openai:<model>"
}
```

## 6. 样本沉淀规则

每次识别可以保存以下元数据：

```text
sample_id
pet_id
device_id
captured_at
image_path
predicted_behavior
confidence
provider
model_version
review_status
final_behavior
```

建议规则：

```text
confidence >= 0.85  进入训练候选集
0.50 <= confidence < 0.85  进入待确认
confidence < 0.50  记录为 uncertain，不直接训练
用户手动修正后的样本  优先进入训练集
```

## 7. 训练节奏

不做逐帧实时训练，改成批量训练：

```text
1. 每天定时训练一次，或每类新增 500 张以上再训练
2. 训练完成后先在验证集评估
3. 新模型准确率达标后再切换到 CV_PROVIDER=local
4. 保留上一个可用模型，方便回滚
```

第一阶段本地模型可以先用 ResNet50 做图片行为分类；后续再结合 YOLO 识别宠物、饭盆、水盆、猫砂盆、床等目标位置，提高行为判断稳定性。

## 8. 阶段拆分

第一步：跑通抽帧识别。

```text
video://loop
  -> 后端取最新帧
  -> cv-service placeholder/openai
  -> 后端行为事件落库
```

第二步：沉淀训练数据。

```text
保存图片
保存标签和置信度
区分高置信度和待确认样本
```

第三步：训练本地模型。

```text
整理 dataset/behavior-classifier
训练 ResNet50
评估模型
切换 local provider
```

第四步：接真实摄像头。

```text
RTSP 摄像头接入
真实家庭场景持续采样
优化行为稳定帧规则
```
