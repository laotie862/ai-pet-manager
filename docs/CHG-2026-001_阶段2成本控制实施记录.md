# AI 宠物管家 - CHG-2026-001 阶段2成本控制实施记录

日期：2026-06-09

## 目标

阶段2的目标是把“每 5 秒抽帧识别”的 MVP 链路改成可长期运行的低成本链路：减少无意义 API 调用，限制样本图片膨胀，并补上本地图片生命周期清理。

## 已实现内容

### 1. 识别间隔改为 30 秒

- 默认调度间隔从 `5000ms` 调整为 `30000ms`。
- 已同步：
  - `demo/backend/src/main/resources/application.properties`
  - `.env`
  - `.env.example`
  - `docker-compose.yml`

关键配置：

```properties
PETCARE_BEHAVIOR_ANALYSIS_INTERVAL_MS=30000
```

### 2. 静态画面跳过

新增 `BehaviorFrameChangeDetector`，将 JPEG 帧缩小成 `32x18` 灰度签名，再计算当前帧与上一帧的平均绝对差异。

识别决策规则：

- 首帧必须识别。
- 小于最小 API 间隔时跳过。
- 画面变化低于阈值时跳过。
- 即使一直静态，超过最大静默时间也强制识别一次。

关键配置：

```properties
PETCARE_BEHAVIOR_STATIC_SKIP_ENABLED=true
PETCARE_BEHAVIOR_MIN_API_INTERVAL_SECONDS=30
PETCARE_BEHAVIOR_MAX_STATIC_API_INTERVAL_SECONDS=300
PETCARE_BEHAVIOR_MOTION_THRESHOLD=0.05
```

### 3. 行为变化优先识别

实现方式：

- 后端先用帧差判断“画面是否明显变化”。
- 画面变化达到阈值并满足最小 API 间隔后，调用百炼 API。
- API 返回的新标签继续进入原有稳定帧状态机。
- 稳定帧确认后，才切换行为事件并写入行为统计。

也就是说，“不同”的判断分两层：

1. 图像层不同：帧差超过阈值。
2. 行为层不同：AI 返回的标签稳定变化。

### 4. 样本保存限流

新增每设备、每标签的样本限流，避免一个睡觉画面连续保存大量图片。

默认规则：

- 同一 `deviceId + label` 10 分钟内最多保存 1 张。
- 同一 `deviceId + label` 每天最多保存 300 张。

关键配置：

```properties
PETCARE_BEHAVIOR_SAMPLE_MIN_INTERVAL_SECONDS=600
PETCARE_BEHAVIOR_MAX_SAMPLES_PER_DEVICE_LABEL_PER_DAY=300
```

### 5. 图片生命周期清理

新增 `BehaviorStorageCleanupJob`，每天凌晨 03:20 清理本地过期文件。

默认规则：

- `dataset/behavior-classifier/_pending_review` 保留 30 天。
- `dataset/extracted-frames` 保留 7 天。

关键配置：

```properties
PETCARE_BEHAVIOR_CLEANUP_ENABLED=true
PETCARE_BEHAVIOR_PENDING_REVIEW_RETENTION_DAYS=30
PETCARE_BEHAVIOR_EXTRACTED_FRAMES_PATH=/data/extracted-frames
PETCARE_BEHAVIOR_EXTRACTED_FRAMES_RETENTION_DAYS=7
```

## 数据库检查

阶段2不新增业务表，不需要新增 Flyway 迁移。

原因：

- 静态跳过、API 调用节流、样本保存限流属于运行时策略。
- 样本保存仍沿用 `t_behavior_sample`。
- 行为事件仍沿用 `t_behavior_event`。
- 行为汇总仍沿用 `t_behavior_summary`。

当前应保持 Flyway 到 V5：

```text
V1__init_schema.sql
V2__auth_refresh_tokens.sql
V3__device_stage3.sql
V4__behavior_stage4.sql
V5__behavior_samples.sql
```

Docker Desktop 启动后建议执行：

```powershell
docker compose exec -T postgres psql -U petcare -d petcare -c "select version, description, success from flyway_schema_history order by installed_rank;"
docker compose exec -T postgres psql -U petcare -d petcare -c "\dt"
```

## 验收方式

1. 启动服务：

```powershell
docker compose up -d --build
```

2. 检查后端和 CV：

```powershell
Invoke-RestMethod http://localhost:8080/api/health
Invoke-RestMethod http://localhost:8000/cv/health
```

3. 使用 `video://loop` 视频源启动设备流。

4. 观察效果：

- 静态画面不会持续调用百炼 API。
- 画面明显变化后会触发新一轮识别。
- 行为样本不会高频写入。
- 标签目录仍按 `dataset/behavior-classifier/{label}/yyyy/MM/dd` 存放训练候选图。
