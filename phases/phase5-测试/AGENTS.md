# Phase 5 — 测试

> **周期**：阶段结束后各有测试，最终集成测试 3-4 周 | **输入**：各阶段可运行系统 | **输出物**：测试报告

---

## 一、测试策略

| 阶段 | 测试类型 | 重点 | 工具 |
|:---|:---|:---|:---|
| 4-1 MVP | 单元 + API + CV 准确率 | 核心闭环 | JUnit5, Postman, 人工标注 |
| 4-2 基础版 | 功能 + 兼容性 + 性能 | App 双端 + 推送 | Flutter Test, XCUITest/Espresso |
| 4-3 VIP | 集成 + 支付 + 安全 | 支付链路 + 视频通话 | Mock 沙箱, OWASP ZAP |
| 4-4 内容 | 回归 + 压力 | 资讯流 + 性能 | JMeter, 全量用例回跑 |

---

## 二、测试环境

| 环境 | 配置 | 用途 |
|:---|:---|:---|
| dev | 单机 Docker Compose | 开发自测 |
| test | 1 台应用 + 1 台数据 + 1 台 GPU | 集成测试 |
| staging | 等同生产配置 | 预发布验证 |

### 3.2 性能测试

| 场景 | 指标 | 工具 |
|:---|:---|:---|
| 接口并发 | 1000 QPS, RT < 500ms (P95) | JMeter |
| 数据库 | 写 RT < 200ms, 读 RT < 100ms | pg_stat_statements |
| App 首屏 | 冷启动 < 3s, 首页数据加载 < 1.5s | Firebase Performance |
| 视频流 | 端到端延迟 < 5s | Wireshark |
| GPU 推理 | 单帧推理 < 200ms, 10路并发 < 500ms/帧 | Python profiling |

### 3.3 安全测试

| 项 | 检查点 | 工具 |
|:---|:---|:---|
| 认证 | 无 Token 访问 401, Token 过期 401, 黑名单生效 | Postman |
| 授权 | 普通用户调 VIP 接口 403, 管理员接口权限隔离 | Postman |
| 注入 | SQL 注入, XSS, CSRF, SSRF | OWASP ZAP |
| 数据 | 敏感字段加密存储确认 (phone, rtsp_password) | DB 直查 |
| 支付 | 回调验签, 金额篡改防护, 并发支付 | 代码审查 + Mock |

## 四、Bug 分级与流程

| 等级 | 定义 | 举例 | 处理 |
|:---:|:---|:---|:---|
| P0 | 核心流程阻断 | App 崩溃, 支付失败钱已扣, 登录不了 | 2h 内修复 |
| P1 | 功能不可用 | 告警不发, 摄像头无画面, 日报不生成 | 24h 内修复 |
| P2 | 功能异常但可绕过 | 图表显示错误, PDF 排版问题 | 下个迭代 |
| P3 | 体验优化 | 文案错别字, UI 错位 | 排期修复 |
